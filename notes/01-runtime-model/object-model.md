# 对象模型

Python 对象模型的核心不是“语法规则很多”，而是一条非常统一的原则：

**程序运行时存在的是对象；名字只是绑定到对象的标签；容器保存的也是对象引用；赋值默认不复制对象。**

只要抓住这条线，很多看似零散的 Python “坑”都会消失。它们不是 special case，而是同一个模型在不同场景下的自然结果。

## 1. 对象由身份、类型和值组成

在 Python 中，几乎所有运行时实体都是对象：

- 整数、浮点数、字符串、列表、字典是对象；
- 函数是对象；
- 类是对象；
- 模块是对象；
- 异常是对象；
- `None`、`True`、`False` 也是对象。

一个对象可以从三个维度理解：

```text
object = identity + type + value
```

- `identity`：对象身份。对象一旦创建，身份在生命周期内保持不变。
- `type`：对象类型。决定对象支持哪些操作。
- `value`：对象的值或状态。对可变对象来说，值可以在原对象内部改变；对不可变对象来说，值不能在原对象内部改变。

例如：

```python
x = [1, 2, 3]

print(id(x))      # 身份
print(type(x))    # 类型：list
print(x)          # 值：[1, 2, 3]
```

需要区分两件事：

```python
x = [1, 2]
y = [1, 2]

print(x == y)  # True，值相等
print(x is y)  # False，不是同一个对象
```

`==` 问的是“值是否相等”，`is` 问的是“是否是同一个对象”。这不是语法细节，而是对象模型的根问题。

`id(obj)` 返回对象身份的整数表示。CPython 中它常常和内存地址有关，但这只是实现细节。写 Python 程序时，应把 `id` 理解成“运行时给对象的唯一身份标识”。

## 2. 名字不是盒子，而是绑定

很多误解来自把变量想象成 C/C++ 里的“盒子”：

```text
错误直觉：变量是一个盒子，值被放进盒子里。
Python 模型：对象在内存中，名字绑定到对象。
```

看这段代码：

```python
a = [1, 2, 3]
b = a
```

真实发生的不是“把 `a` 的列表复制给 `b`”，而是：

```text
name a ─┐
        ├──> list object [1, 2, 3]
name b ─┘
```

所以：

```python
a = [1, 2, 3]
b = a

b.append(4)

print(a)  # [1, 2, 3, 4]
print(b)  # [1, 2, 3, 4]
```

这不是 `b` “影响了” `a`。更准确地说，`a` 和 `b` 本来就绑定到同一个列表对象；`append` 修改的是那个列表对象本身。

赋值语句做的是绑定或重新绑定：

```python
x = [1, 2, 3]
y = x

x = [4, 5, 6]

print(y)  # [1, 2, 3]
print(x)  # [4, 5, 6]
```

图像上是：

```text
一开始：

x ─┐
   ├──> [1, 2, 3]
y ─┘

x = [4, 5, 6] 之后：

x ───> [4, 5, 6]
y ───> [1, 2, 3]
```

重新绑定名字不会修改旧对象，只是让这个名字指向另一个对象。

这也解释了一个关键事实：

**Python 中变量本身没有类型，变量绑定的对象才有类型。**

```python
x = 1
print(type(x))  # int

x = "hello"
print(type(x))  # str
```

不是变量 `x` 的类型改变了，而是名字 `x` 从一个 `int` 对象重新绑定到了一个 `str` 对象。

## 3. 函数传参也是名字绑定

Python 的函数调用也不需要另造一套规则。参数传递依然是名字绑定。

```python
def mutate(xs):
    xs.append(3)

a = [1, 2]
mutate(a)
print(a)  # [1, 2, 3]
```

调用 `mutate(a)` 时，函数局部名字 `xs` 绑定到 `a` 所绑定的同一个列表对象：

```text
outer name a ─┐
              ├──> [1, 2]
local name xs ─┘
```

所以 `xs.append(3)` 修改的是共享的列表对象。

但是重新绑定参数名不会影响外部名字：

```python
def rebind(xs):
    xs = [9, 9, 9]

a = [1, 2]
rebind(a)
print(a)  # [1, 2]
```

因为 `xs = [9, 9, 9]` 只是让局部名字 `xs` 绑定到新列表，不会改变外部名字 `a` 的绑定。

这说明 Python 不是简单的“传值”或“传引用”二分。更精确的说法是：

**函数调用把实参对象绑定到形参名字；对象是否会被观察到改变，取决于函数内部是否修改了共享对象本身。**

## 4. 容器保存的是引用，不是对象副本

列表、元组、字典、集合等容器保存的是对象引用。这个事实非常重要。

```python
a = [1, 2]
b = [a, a]

print(b)  # [[1, 2], [1, 2]]

a.append(3)

print(b)  # [[1, 2, 3], [1, 2, 3]]
```

`b` 不是保存了两个独立列表，而是保存了两次对同一个列表对象的引用：

```text
b ───> [ ref ─┐, ref ─┐ ]
              │       │
              └──> [1, 2]
```

这也解释了一个经典现象：

```python
rows = [[0] * 3] * 3
rows[0][0] = 1

print(rows)
# [[1, 0, 0], [1, 0, 0], [1, 0, 0]]
```

`[inner] * 3` 复制的是引用，不是递归复制内部列表。三个位置指向同一个内部列表。

正确写法通常是：

```python
rows = [[0] * 3 for _ in range(3)]
rows[0][0] = 1

print(rows)
# [[1, 0, 0], [0, 0, 0], [0, 0, 0]]
```

列表推导式每次循环都会创建一个新的内部列表对象。

## 5. 可变与不可变：改变对象，还是创建新对象

Python 中常见不可变对象包括：

- `int`
- `float`
- `bool`
- `str`
- `bytes`
- `tuple`
- `frozenset`
- `None`

常见可变对象包括：

- `list`
- `dict`
- `set`
- `bytearray`
- 大多数自定义类实例

可变和不可变的本质区别是：

```text
可变对象：可以在保持身份不变的情况下改变值。
不可变对象：值不能在原对象内部改变；看似修改，实际是创建新对象并重新绑定名字。
```

例如列表：

```python
xs = [1, 2]
before = id(xs)

xs.append(3)

after = id(xs)
print(xs)             # [1, 2, 3]
print(before == after)  # True
```

`append` 修改的是原列表，身份不变。

而字符串：

```python
s = "py"
before = id(s)

s = s + "thon"

after = id(s)
print(s)              # python
print(before == after)  # False
```

字符串不能原地修改。`s + "thon"` 创建了一个新字符串，然后名字 `s` 重新绑定到新对象。

同理：

```python
x = 1
x += 1
```

不是把整数对象 `1` 改成 `2`，而是创建或取得整数对象 `2`，然后让名字 `x` 绑定到它。

## 6. `+=` 的统一理解

`+=` 容易让人误以为有特殊规则。它的统一理解是：

**对象如果支持原地修改，`+=` 倾向于修改原对象；否则创建新对象并重新绑定名字。**

列表是可变对象：

```python
a = [1, 2]
b = a

a += [3]

print(a)       # [1, 2, 3]
print(b)       # [1, 2, 3]
print(a is b)  # True
```

字符串是不可变对象：

```python
a = "py"
b = a

a += "thon"

print(a)       # python
print(b)       # py
print(a is b)  # False
```

这里没有两套世界观。差别只在于：左侧对象是否能在原地改变。

一个更微妙的例子是元组：

```python
t = ([1, 2],)
t[0].append(3)

print(t)  # ([1, 2, 3],)
```

元组不可变，意思是“元组内部保存的引用不能换掉”，不是“引用指向的对象都不能变”。

所以：

```python
t = ([1, 2],)
t[0] = [9, 9]       # TypeError
t[0].append(3)      # 可以
```

元组的第一个槽位仍然指向同一个列表对象；只是那个列表对象自身变了。

这叫“浅层不可变”。理解容器保存引用之后，它就不再反直觉。

## 7. 身份、相等性与单例

`is` 判断身份：

```python
a is b
```

含义是：`a` 和 `b` 是否绑定到同一个对象。

`==` 判断相等性：

```python
a == b
```

含义是：两个对象的值是否按其类型定义的规则相等。

通常应该这样用：

```python
if x is None:
    ...

if name == "Alice":
    ...
```

`None` 是单例对象，用 `is None` 判断身份最准确。字符串、整数、列表等普通值比较应该用 `==`。

不要用 `is` 比较数字或字符串的值：

```python
a = 256
b = 256
print(a is b)  # 在某些实现或场景中可能是 True

x = 10000
y = 10000
print(x is y)  # 不应该依赖结果
```

CPython 可能缓存小整数或驻留某些字符串。这是性能优化，不是语言语义。把实现优化当成语义规则，是 shallow understanding 的典型来源。

正确心智模型是：

```text
值相等：用 ==
同一对象：用 is
单例判断：用 is None / is True / is False
```

其中 `is True` 和 `is False` 也要谨慎使用。大多数条件判断应直接写：

```python
if flag:
    ...
```

而不是：

```python
if flag is True:
    ...
```

因为很多对象可以在布尔上下文中为真或假，但它们并不是 `True` 这个单例对象。

## 8. 浅拷贝：复制外壳，共享内部对象

浅拷贝创建一个新的外层容器，但不会递归复制内部元素。内部元素仍然引用原来的对象。

```python
import copy

a = [[1, 2], [3, 4]]
b = copy.copy(a)

print(a is b)        # False，外层列表不同
print(a[0] is b[0])  # True，内层列表共享

b[0].append(99)

print(a)  # [[1, 2, 99], [3, 4]]
print(b)  # [[1, 2, 99], [3, 4]]
```

图像是：

```text
a ───> [ ref ─────┐, ref ─────┐ ]
                  │           │
                  v           v
                [1, 2]      [3, 4]
                  ^           ^
                  │           │
b ───> [ ref ─────┘, ref ─────┘ ]
```

常见浅拷贝方式：

```python
b = a.copy()
b = list(a)
b = a[:]
b = copy.copy(a)
```

对字典也类似：

```python
d1 = {"user": {"name": "Alice"}}
d2 = d1.copy()

d2["user"]["name"] = "Bob"

print(d1)  # {'user': {'name': 'Bob'}}
```

外层字典复制了，内层字典没有复制。

浅拷贝不是“不完整的拷贝”，它有明确语义：

**创建新的外层对象，并复用其中已有元素引用。**

当元素是不可变对象时，浅拷贝通常足够：

```python
# 不可变元素
a = [1, 2, 3]
b = a.copy()

b[0] += 100

print(a)  # [1, 2, 3]
print(b)  # [101, 2, 3]
```

因为整数对象本身不会被修改。这里实际上是创建一个新的 int 对象，再将 `b[0]` 重新绑定到这个新 `int` 对象。

## 9. 深拷贝：复制对象图

深拷贝会递归复制对象引用到的内部对象：

```python
import copy

a = [[1, 2], [3, 4]]
b = copy.deepcopy(a)

print(a is b)        # False
print(a[0] is b[0])  # False

b[0].append(99)

print(a)  # [[1, 2], [3, 4]]
print(b)  # [[1, 2, 99], [3, 4]]
```

更准确地说，`deepcopy` 不是“无脑递归复制所有东西”，而是在复制对象图：

```text
对象不是孤立点，而是一张由引用连接成的图。
浅拷贝复制一层节点。
深拷贝尝试复制整张可达对象图。
```

它还需要处理共享引用和循环引用。

例如共享引用：

```python
import copy

inner = [1, 2]
a = [inner, inner]
b = copy.deepcopy(a)

print(b[0] is b[1])  # True
print(b[0] is inner) # False
```

`deepcopy` 会保留“两个位置指向同一个内部对象”这个结构关系，只是内部对象本身换成了复制品。这依靠内部的 memo 机制完成。

循环引用也类似：

```python
import copy

a = []
a.append(a)  # self-referential structure, [[...]]

b = copy.deepcopy(a)    # deepcopy 很聪明, 不会无限递归

print(b is b[0])  # True
print(a is b)     # False
```

所以深拷贝的正确理解是：

**在尽量保持原对象图拓扑关系的前提下，构造一张新的对象图。**

深拷贝不是越多越好。它可能昂贵，也可能复制了不该复制的资源，比如文件句柄、锁、数据库连接、大型缓存。工程中更常见的策略是：

- 明确哪些数据需要隔离；
- 对不可变数据放心共享；
- 对可变数据控制所有权；
- 必要时用浅拷贝复制边界；
- 只有确实需要独立对象图时才用深拷贝。

## 10. 默认参数陷阱不是陷阱

虽然默认参数属于函数机制，但它和对象模型关系极深。经典例子：

```python
def f(x=[]):
    x.append(1)
    return x

print(f())  # [1]
print(f())  # [1, 1]
print(f())  # [1, 1, 1]
```

肤浅理解会说：“Python 有一个可变默认参数陷阱。”

深入理解应该说：

**函数定义执行时创建默认参数对象；之后每次省略参数调用，形参名字都绑定到同一个默认对象。**

也就是：

```text
function object f
  └── default x ───> []
```

每次调用 `f()` 时，局部名字 `x` 都绑定到这个列表对象。`x.append(1)` 修改的是同一个列表。

推荐写法是：

```python
def f(x=None):
    if x is None:
        x = []
    x.append(1)
    return x
```

这里 `None` 不是“魔法补丁”，而是一个有意选择的哨兵对象：

- `None` 是不可变单例；
- 可以用 `is None` 精确判断是否传入参数；
- 每次需要默认列表时，在函数调用内部创建新列表。

这就把“共享默认对象”变成“每次调用创建新对象”。

## 11. 不可变对象带来共享，可变对象需要所有权意识

不可变对象的一个巨大价值是：它们可以安全共享。

```python
a = "hello"
b = a
```

`a` 和 `b` 共享同一个字符串对象没有问题，因为任何一方都不能原地修改它。

可变对象则需要考虑 aliasing：

```python
config = {"debug": False}
a = config
b = config

a["debug"] = True

print(b["debug"])  # True
```

这里的问题不是 Python 奇怪，而是共享可变状态本来就需要所有权设计。

在 C++ 中你会问：

```text
这个对象由谁拥有？
这里是复制、借用、共享，还是移动？
```

在 Rust 中你会问：

```text
这里是所有权转移、不可变借用，还是可变借用？
```

在 Python 中也应该问类似问题，只是语言不会强制你在类型系统里写出来：

```text
这个函数会不会修改传入对象？
调用者是否还会继续使用这个对象？
是否需要复制边界？
是否可以改用不可变表示？
```

例如一个函数如果会修改参数，最好从命名或文档上表现出来：

```python
def normalize_in_place(xs: list[float]) -> None:
    total = sum(xs)
    for i, x in enumerate(xs):
        xs[i] = x / total
```

如果不想修改调用者的数据，就创建新对象：

```python
def normalized(xs: list[float]) -> list[float]:
    total = sum(xs)
    return [x / total for x in xs]
```

对象模型没有替你消灭共享可变状态，它只是让共享可变状态非常直接地暴露出来。

## 12. 对象生命周期：引用存在，对象就可能存在

Python 程序员通常不手动管理内存，但对象生命周期依然由引用关系决定。

```python
x = [1, 2, 3]
y = x

del x

print(y)  # [1, 2, 3]
```

`del x` 删除的是名字 `x` 的绑定，不是强行销毁对象。因为 `y` 仍然引用那个列表，对象还活着。

```python
x = [1, 2, 3]
del x
```

如果没有其他引用指向这个列表，它就可以被垃圾回收。

需要注意：

```python
del obj
```

含义不是“析构这个对象”，而是“删除当前作用域中名字 `obj` 的绑定”。对象是否销毁，取决于是否还有其他引用。

CPython 主要使用引用计数，并辅以循环垃圾回收；其他 Python 实现可能不同。工程心智模型应停留在语言层：

**当对象不再可达时，它可以被回收；不要依赖具体回收时机来写业务逻辑。**

如果需要确定性释放外部资源，使用上下文管理器：

```python
with open("data.txt") as f:
    text = f.read()
```

这不是对象模型的例外，而是因为“内存对象生命周期”和“外部资源生命周期”不是同一个问题。文件句柄、网络连接、锁等资源需要显式的协议来管理。

## 13. 从对象模型看常见代码现象

### 现象一：修改列表元素

```python
xs = [1, 2, 3]
ys = xs

xs[0] = 99

print(ys)  # [99, 2, 3]
```

解释：`xs` 和 `ys` 绑定到同一个列表；`xs[0] = 99` 修改列表对象内部保存的第一个引用。

### 现象二：给列表变量重新赋值

```python
xs = [1, 2, 3]
ys = xs

xs = [99, 2, 3]

print(ys)  # [1, 2, 3]
```

解释：`xs = ...` 重新绑定名字，不修改旧列表。

### 现象三：排序

```python
xs = [3, 1, 2]
ys = xs

xs.sort()

print(ys)  # [1, 2, 3]
```

解释：`list.sort()` 原地修改列表。

```python
xs = [3, 1, 2]
ys = xs

xs = sorted(xs)

print(ys)  # [3, 1, 2]
print(xs)  # [1, 2, 3]
```

解释：`sorted(xs)` 创建新列表，`xs = ...` 重新绑定名字。

### 现象四：函数里 `+=`

```python
def add_item(xs):
    xs += [1]

a = []
add_item(a)
print(a)  # [1]
```

解释：这里的 `+=` 修改原列表对象。

```python
def add_text(s):
    s += "!"

a = "hi"
add_text(a)
print(a)  # hi
```

解释：字符串不可变，`s += "!"` 创建新字符串并重新绑定局部名字 `s`。

## 14. 一个统一检查表

遇到任何“Python 这里为什么这样”的问题，先不要背 special case。按下面顺序问：

1. 这里有哪些对象？
2. 哪些名字绑定到哪些对象？
3. 哪些容器保存了哪些对象引用？
4. 这一步是修改对象，还是重新绑定名字？
5. 对象是可变还是不可变？
6. 如果发生复制，是浅拷贝还是深拷贝？
7. 我关心的是身份 `is`，还是值相等 `==`？

例如：

```python
a = [[1], [2]]
b = a.copy()
c = a[0]

b[0].append(99)
c.append(100)

print(a)
print(b)
```

用对象图理解：

- `a.copy()` 只复制外层列表；
- `a[0]`、`b[0]`、`c` 都指向同一个内部列表；
- 两次 `append` 都修改这个内部列表。

所以：

```python
print(a)  # [[1, 99, 100], [2]]
print(b)  # [[1, 99, 100], [2]]
```

这不是特殊情况。这只是对象、名字、引用、可变性、浅拷贝共同作用的结果。

## 15. 最小心智模型

可以把 Python 对象模型压缩成几句话：

1. 一切运行时实体都是对象。
2. 对象有身份、类型和值。
3. 名字绑定到对象；赋值是绑定，不是复制。
4. 函数传参是把对象绑定到局部参数名。
5. 容器保存的是对象引用。
6. 可变对象能在身份不变的情况下改变值。
7. 不可变对象的“修改”会产生新对象并重新绑定名字。
8. `is` 比较身份，`==` 比较值。
9. 浅拷贝复制外层对象，深拷贝复制对象图。
10. 多数 Python “坑”都不是例外，而是这个模型的直接推论。

真正的目标不是记住更多规则，而是让规则变少。理解越深，Python 越简单。
